1需求背景

本功能基于 e 签宝实现企业实名认证和授权。商城侧提供【个人认证】【企业认证】两个入口，用户选择后进入对应流程。后端判断企业认证场景后传参给 e 签宝，e 签宝 H5 根据后端参数自动适配首次认证/已认证授权流程。

当前业务流程：
用户 → 物料商城 → 【个人认证】→ e签宝
用户 → 物料商城 → 【企业认证】→ e签宝

2需求目标

目标说明
核心目标实现企业实名认证和电子合同授权功能
业务价值企业认证后可使用账期支付、签署电子合同
关键约束认证和授权必须通过 e 签宝完成

6功能详述

6.1 认证入口页

原型描述：权益引导页，展示认证价值和当前状态。未认证显示【个人认证】【企业认证】两个入口按钮，已认证显示认证状态。

作为一个未认证用户，我想要了解认证价值和流程，以便于决定是否进行认证。

前置条件：用户已登录

核心逻辑：检查用户认证状态 → 点【个人认证】进入个人认证流程 / 点【企业认证】进入企业认证流程

边界条件：已认证通过显示成功状态；已认证待授权显示等待状态；有草稿允许恢复或重新填写

验收标准：

  • ✅ 用户能看到认证价值说明
  • ✅ 未认证用户能看到【个人认证】和【企业认证】两个入口
  • ✅ 已认证用户能看到当前认证状态

6.2 草稿恢复页

原型描述：检测到历史草稿时显示，提供【继续认证】和【重新填写】两个选项。

作为一个有历史草稿的企业用户，我想要快速恢复之前的填写进度，以便于节省时间。

前置条件：用户有未提交的认证草稿

核心逻辑：点【继续认证】回填草稿数据；点【重新填写】清除草稿重新开始

验收标准：

  • ✅ 正确显示草稿创建时间
  • ✅ 继续认证能回填之前数据
  • ✅ 重新填写能清空草稿

6.3 企业信息填写页

原型描述：企查查搜索框，输入≥2字触发模糊搜索，选中企业后自动填充4个字段。

作为一个企业用户，我想要快速填写企业信息，以便于减少手动输入。

前置条件：用户点击【企业认证】入口后，进入企业信息填写页

核心逻辑：企查查搜索 → 选中后自动填充企业名称/统一社会信用代码/法定代表人/注册地址 → 【下一步】进入过渡页

边界条件：搜索无结果提示"未找到企业"；信息不完整【下一步】禁用

验收标准：

  • ✅ 输入≥2字触发搜索
  • ✅ 选中企业后自动填充4个字段
  • ✅ 点击【下一步】进入过渡页

6.4 过渡页

原型描述：显示"正在准备认证材料..."加载动画，纯前端1-2秒，不调接口，自动跳转。

作为一个企业用户，我想要等待认证材料准备，以便于看到完整表单。

前置条件：企业信息已填写，点击【下一步】

核心逻辑：纯前端setTimeout 1-2秒后自动跳转，不调用任何后端接口

验收标准：

  • ✅ 显示加载动画
  • ✅ 1-2秒后自动跳转
  • ✅ 不调接口（纯前端）

6.5 完整认证表单（统一表单）

原型描述：前端展示统一表单页面（PC/小程序各只有一套表单），不做任何场景判断：

  • 场景一（首次认证）：法人身份证号需手动填写
  • 场景二（已认证授权）：法人身份证号由后端自动回填（只读）

表单字段：法人身份证号 + 经办人（姓名/身份证/接收订单消息电话）+ 选填项 + 必勾协议

作为一个企业用户，我想要填写认证信息，以便于完成e签宝实名认证。

前置条件：通过过渡页

核心逻辑：

  • 场景一：用户填写法人身份证号 → 提交后获取authUrl → 跳转e签宝H5
  • 场景二：法人身份证号后端自动回填（只读）→ 提交后获取authUrl → 跳转e签宝授权页

边界条件：法人身份证格式错误表单验证不通过；协议未勾选提交按钮禁用；授权申请最多3次

验收标准：

  • ✅ 场景一：法人身份证号必填
  • ✅ 场景二：法人身份证号自动回填且只读
  • ✅ 协议未勾选提交按钮禁用
  • ✅ 提交后跳转e签宝H5
  • ✅ 场景二第4次提交返回错误提示

7边界条件与异常处理

场景条件处理方式
草稿保存任何环节自动每30秒保存 + 页面失焦保存
草稿有效期永久永久保存，90天未更新自动清理
授权重试最多3次第4次提交返回400
授权超时24小时超时自动失效，需重新申请
认证失败可重新认证点【重新认证】跳转表单并回填数据
场景二表单法人身份证后端自动回填，前端只读

8验收标准

8.1 状态机

状态值状态名含义下一步
UNVERIFIED (0)未认证初始状态点【个人认证】或【企业认证】进入对应流程
DRAFT (1)草稿有未完成的认证草稿【继续认证】回填 / 【重新填写】清空
VERIFYING (2)认证中已提交，等待e签宝回调等待，可点【刷新状态】
REJECTED (4)认证失败展示失败原因点【重新认证】→ 跳表单回填
AUTHZ_PENDING (5)授权待处理等待管理员审批等待 / 点【刷新状态】
AUTHZ_APPROVED (6)授权通过认证完成，可签合同【去签合同】
AUTHZ_REJECTED (7)授权拒绝管理员拒绝点【重新申请】→ 弹窗改理由（最多3次）

3授权超时提示超时，需重新申请
