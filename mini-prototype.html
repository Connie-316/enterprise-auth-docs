微信小程序企业认证原型，3步流程

🏗️ 核心流程
企查查搜索 → 填写表单 → 提交 → 后端判断场景 → 调e签宝接口1 → 返回authUrl → 跳转WebView

后端判断逻辑：
  auth_status = UNVERIFIED(0) → 场景一（businessScene=FIRST_TIME）
  auth_status = 其他         → 场景二（businessScene=ALREADY_AUTH）
🔵 场景一：首次认证
①→②→②.₅→④ → WebView
→ 人脸+营业执照+授权
🔴 场景二：已认证授权
①→②→②.₅→④ 同上
→ WebView（跳过人脸）
⚠️ P0必改项（开发前必读）
1. clientType=MINI_APP
2. redirectUrl指向中间H5页 + wx.miniProgram.navigateTo 跳回
3. 业务域名：微信公众平台配置 openapi.esign.cn
4. X-Platform: miniapp

⚠️ 后端开发必须修改的3项（P0）
1. clientType 动态识别：读取请求Header的 X-Platform 字段，小程序端传 MINI_APP，H5端传 H5
2. redirectUrl 适配：小程序端的 redirectUrl 应指向部署在业务域名下的中间H5页（miniapp-callback.html），由该页面调用 wx.miniProgram.navigateTo 跳回小程序
3. 业务域名配置：小程序后台配置 webviewUrl 业务域名，否则 web-view 组件无法加载 e签宝 H5 页面

🔗 e签宝接口调用时序（小程序端）
① 小程序前端 POST /api/company/auth/submit Header: { Authorization, X-Platform: miniapp } ② 商城后端 获取AccessToken (HMAC-SHA256鉴权) POST /v3/oauth2/access_token → 缓存Redis (有效期7200秒) ③ 商城后端 调用e签宝接口1 (获取认证页面链接) POST /v3/org-auth/create clientType=MINI_APP ← 关键！ redirectUrl=https://xxx.com/auth/miniapp-callback.html notifyUrl=https://xxx.com/api/company/auth/callback ④ 返回 authUrl → 前端 WebView 加载 ⑤ 用户在WebView完成认证 → e签宝跳转redirectUrl(中间H5页) ⑥ 中间H5页 wx.miniProgram.navigateTo → 跳转小程序结果页 ⑦ e签宝异步回调 notifyUrl → 后端验签(HMAC-SHA256) → 查询接口2 → 更新数据库
